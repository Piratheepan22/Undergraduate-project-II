Week 10

Overview

This week, I developed a full-stack web application to facilitate real-time testing and visualization of the AI-generated damage maps. The application uses Flask as the backend server to handle model inference and JavaScript on the frontend to create an interactive user interface. This development framework provides an intuitive platform for uploading satellite image pairs, running the trained change detection model, and visualizing the results with side-by-side comparisons of predictions and ground truth annotations. The web application ensures that our system is fully optimized for practical deployment and high-accuracy disaster assessment.

Detailed Update

Web Application Development

With the core model architecture established last week, this week focused on creating a practical deployment platform. The web application bridges the gap between the trained model and end-users, providing an accessible interface for disaster assessment without requiring technical expertise.

Tasks Completed This Week:

  • Designed and implemented Flask backend API for model serving
  • Created RESTful endpoints for image upload and processing
  • Integrated trained change detection model into Flask application
  • Developed frontend interface using HTML, CSS, and JavaScript
  • Implemented interactive image upload with drag-and-drop functionality
  • Created real-time visualization of change detection results
  • Built side-by-side comparison view for predictions vs. ground truth
  • Added overlay visualization with adjustable opacity
  • Implemented result export functionality (download predictions)

Application Architecture

Backend - Flask Server:

The Flask backend handles all computational tasks:

  • Model Loading:
    • Load pre-trained ViT-CNN model on server startup
    • Keep model in memory for fast inference
    • Support GPU acceleration if available
  • Image Processing Pipeline:
    • Receive uploaded before/after images
    • Validate image format and dimensions
    • Apply preprocessing (resize, normalize)
    • Create 6-channel input tensor
  • Model Inference:
    • Run change detection model on preprocessed input
    • Post-process output (thresholding, cleanup)
    • Generate visualization-ready change mask
  • Response Handling:
    • Convert predictions to image format
    • Return results as JSON with base64 encoded images
    • Include confidence scores and metadata

Frontend - Interactive Interface:

The JavaScript-powered frontend provides intuitive interaction:

  • Image Upload Interface:
    • Drag-and-drop zone for before/after images
    • File browser with format validation
    • Image preview before processing
    • Clear/reset functionality
  • Visualization Components:
    • Side-by-side view: Original images + prediction
    • Overlay mode: Change mask overlaid on images
    • Opacity slider for overlay adjustment
    • Zoom and pan controls for detailed inspection
  • Results Display:
    • Color-coded change visualization (red for changes)
    • Statistics: Changed area percentage, pixel counts
    • Confidence map visualization
    • Comparison with ground truth if available
  • User Controls:
    • Process button to trigger inference
    • Download button to save predictions
    • Toggle between different visualization modes
    • Progress indicators during processing

Key Features Implemented

1. Real-Time Processing:

  • Upload images and receive results within seconds
  • Asynchronous processing to keep UI responsive
  • Loading indicators and progress feedback

2. Interactive Visualization:

  • Multiple view modes for different analysis needs
  • Adjustable opacity overlays for change emphasis
  • Zoom functionality for detailed inspection
  • Toggle between RGB and change mask views

3. Comparison Tools:

  • Side-by-side display of before/after/prediction
  • Ground truth overlay for accuracy assessment
  • Difference highlighting between prediction and ground truth
  • Quantitative metrics display (IoU, F1-score)

4. Export Capabilities:

  • Download predicted change masks
  • Save comparison visualizations
  • Export results with metadata (coordinates, timestamps)
  • Batch processing support for multiple image pairs
