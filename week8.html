Week 8

Overview

This week, I developed an automated workflow using Google Colab to process the collected MSI triplets and generate change detection masks using spectral indices. The workflow validates Sentinel-2 data, computes NDVI and NDWI indices, applies automated thresholding to identify changes, and distinguishes between persistent and transient changes. This automated approach provides a systematic way to process all collected triplets and generates standardized outputs for comparison with manual annotations.

Detailed Update

Automated Change Detection Workflow

Building on last week's spectral index exploration, I created a complete processing pipeline that automatically analyzes MSI triplets and generates change masks. This workflow serves as both a validation tool for our data and a baseline method for automated change detection.

Tasks Completed This Week:

  • Developed Google Colab notebook for automated MSI triplet processing
  • Implemented data validation to verify triplet completeness and band integrity
  • Created RGB and false-color composite visualizations for quick quality checks
  • Computed NDVI and NDWI for all three temporal phases (Before, During, After)
  • Generated delta-index maps showing changes relative to Before image
  • Applied Otsu's automatic thresholding method to identify significant changes
  • Implemented morphological operations for mask cleanup and noise reduction
  • Created persistent vs. transient change classification logic
  • Exported results in multiple formats: GeoTIFF (with geospatial info) and PNG (for visualization)
  • Tested workflow on sample triplets from different disaster types

Workflow Components

1. Data Loading and Validation

  • Load 13-band Sentinel-2 GeoTIFF files for Before, During, and After phases
  • Verify band completeness and correct ordering
  • Check spatial alignment and coordinate reference systems
  • Validate pixel value ranges and identify any corrupted data

2. Visualization and Quality Check

  • RGB Composites: True color visualization (Red-Green-Blue)
  • False-Color Composites: NIR-Red-Green for vegetation emphasis
  • Band Inspection Grids: Display selected individual bands across time
  • Quick visual assessment to identify obvious issues or cloud contamination

3. Spectral Index Computation

Calculate indices for each timestamp:

// NDVI: Vegetation index

NDVI = (Band8 - Band4) / (Band8 + Band4)

NDVI = (NIR - Red) / (NIR + Red)


// NDWI: Water/moisture index

NDWI = (Band3 - Band8) / (Band3 + Band8)

NDWI = (Green - NIR) / (Green + NIR)

4. Change Detection via Delta Maps

Compute change by subtracting Before index from other phases:

// Compute deltas relative to Before

Δ_During = Index_During - Index_Before

Δ_After = Index_After - Index_Before


// Large absolute deltas indicate change

Change_During = |Δ_During| > threshold

Change_After = |Δ_After| > threshold

5. Otsu's Automatic Thresholding

Instead of manually selecting thresholds, I used Otsu's method, which automatically determines the optimal threshold by analyzing the histogram of delta values:

  • Separates changed pixels from unchanged pixels based on statistical distribution
  • Adapts to different scenes and change magnitudes automatically
  • No manual tuning required for different locations
  • Particularly effective when change and no-change pixels form distinct populations

6. Morphological Cleanup

Raw thresholded masks often contain noise and small isolated pixels. Morphological operations clean these up:

  • Opening: Removes small isolated noise pixels
  • Closing: Fills small holes within change regions
  • Area filtering: Removes very small connected components
  • Results in cleaner, more interpretable change masks

7. Persistent vs. Transient Change Classification

One of the key innovations this week is separating two types of changes:

  • Persistent Changes:
    • Present in BOTH During and After images
    • Represents lasting damage (permanent flooding, landslide scars)
    • Logic: Mask_Persistent = Mask_During AND Mask_After
  • Transient Changes:
    • Present in During but NOT in After images
    • Represents temporary conditions (receded floodwaters, temporary debris)
    • Logic: Mask_Transient = Mask_During AND (NOT Mask_After)

This classification provides valuable information about change dynamics and helps understand which impacts were temporary versus permanent.

Output Generation

The workflow generates multiple output types for each triplet:

  • Index Rasters: NDVI and NDWI for Before, During, After (6 GeoTIFFs)
  • Delta Maps: Δ_During and Δ_After for both indices (4 GeoTIFFs)
  • Binary Masks: Thresholded change masks (GeoTIFF format)
  • Classified Masks: Persistent vs. Transient change separation
  • Visualizations: PNG format for easy viewing and documentation
  • Composites: RGB and false-color for context

Key Advantages of This Approach

  • Fully Automated: No manual intervention required after initial setup
  • Reproducible: Same processing applied consistently to all triplets
  • Adaptive: Otsu's method adjusts to different scenes automatically
  • Multi-index: NDVI and NDWI provide complementary change signals
  • Temporal Intelligence: Distinguishes persistent from transient changes
  • Baseline Method: Provides comparison point for manual annotations

Observations and Findings

  • False-color composites (NIR-Red-Green) make vegetation and water changes much more visible than RGB
  • NDVI excels at detecting vegetation loss from landslides
  • NDWI performs better for flood extent mapping
  • Otsu thresholding works well when change is significant relative to background variation
  • Morphological cleanup is essential for removing sensor noise and small artifacts
  • Persistent change masks often correspond well with manually annotated damage areas
  • Transient changes help identify temporary flooding that receded before the After image

Workflow Testing Results

Tested the workflow on several triplets representing different disaster scenarios:

  • Flood scenarios: NDWI delta maps clearly showed water extent changes
  • Landslide areas: NDVI delta maps highlighted vegetation loss
  • Mixed impacts: Combination of both indices captured complex changes
  • Automated masks showed good agreement with visual interpretation
  • Processing time: ~2-3 minutes per triplet on Google Colab
