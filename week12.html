Week 12

Overview

This week involved comprehensive dataset auditing and expansion to broaden the scope beyond flood-focused imagery. I conducted a thorough review of our current dataset coverage across Sri Lanka, mapped spatial distribution patterns, and expanded to include tsunami and wildfire disasters with bi-temporal image pairs and binary change masks. Additionally, I developed event-specific GEE export templates tailored to the temporal dynamics of different disaster types, and studied change detection research papers to improve dataset structure and documentation quality for future release.

Detailed Update

Dataset Audit and Coverage Analysis

With significant data collected over previous weeks, this week began with a systematic audit to assess current coverage, identify gaps, and plan strategic expansion.

Current Dataset Status:

  • Total Images: 272 images collected across Sri Lanka
  • Primary Focus: Flood events (tri-temporal: Before/During/After)
  • Geographic Spread: Multiple districts with varying density
  • Data Types: RGB imagery, MSI 13-band data, binary masks, spectral indices

Spatial Distribution Analysis:

I created a coverage map showing tile center distribution across Sri Lanka, revealing:

  • Dense zones: Western Province (Colombo, Gampaha) and Central highlands
  • Moderate coverage: Eastern Province (Trincomalee area), North-Central regions
  • Sparse zones: Northern Province, some Southern coastal areas
  • Insight: Coverage aligns with major flood-prone river basins and population centers

Multi-Hazard Dataset Expansion

To increase dataset versatility and research applicability, I expanded beyond floods to include tsunami and wildfire events, shifting to a bi-temporal (Before/After) structure standard in change detection research.

Why Expand to Multi-Hazard:

  • Broader applicability for disaster monitoring systems
  • Enables model generalization across different change types
  • Aligns with standard change detection benchmarks
  • Increases dataset research value and citation potential

Tasks Completed This Week:

  • Analyzed coverage density and identified underrepresented regions
  • Expanded dataset scope from flood-only to multi-hazard (tsunami, wildfire)
  • Transitioned from tri-temporal to bi-temporal structure for new hazard types
  • Developed event-specific GEE export templates for tsunami and wildfire scenarios
  • Collected wildfire image pairs from California 2021 fire season (7 samples)

  • Reviewed change detection research papers for dataset design best practices
  • Documented naming conventions and metadata standards

Event-Specific GEE Templates

Different disaster types have distinct temporal dynamics requiring tailored image selection strategies:

Temporal Windows by Hazard Type:

Event Type BEFORE Window ON/DURING Window AFTER Window
Flood -90 days ±3 days +10 days
Tsunami -120 to -30 days 0 to +10 days (fallback: +30, +60) +30 to +120 days
Wildfire -90 to -14 days 0 to +5 days (fallback: +14, +30) +30 to +120 days

Tsunami Template Considerations:

  • Longer BEFORE window accounts for seasonal coastal variations
  • Extended AFTER window captures recovery and reconstruction
  • Fallback dates handle cloud occlusion during monsoon periods
  • Focus on coastal and low-lying areas

Wildfire Template Considerations:

  • Shorter ON window captures active fire period
  • Haze proxy selection (low blue band values) for smoke-affected scenes
  • SWIR bands particularly valuable for burn scar detection
  • Extended AFTER window shows vegetation recovery

Wildfire Dataset Sample - California 2021

Collected 7 bi-temporal image pairs from California 2021 wildfire season as proof-of-concept:

  • Structure: Before RGB + After RGB + Binary Change Mask
  • Change Type: Burn scars, vegetation loss, infrastructure damage
  • Resolution: 10m Sentinel-2 imagery
  • Mask Generation: Manually annotated burned areas

Research Paper Review Insights

Studied benchmark change detection papers to improve dataset quality and documentation:

Key Takeaways Applied:

  • Clear Triplet Structure: Standardized Before/After/Mask organization
  • Reproducible Splits: Defined train/val/test splits from the start
  • Transparent Metadata: Document acquisition dates, coordinates, preprocessing steps
  • Evaluation Protocols: Establish baseline metrics (IoU, F1-score)
  • Known Limitations: Explicitly state sampling bias and coverage gaps
  • Visual Examples: Include representative samples showing dataset difficulty

Papers Reviewed:

  • WHU Change Detection Dataset: Benchmark dataset organization and presentation style
  • Urban Built-Up Area Detection: Temporal comparison framing for land-cover change
  • Both papers emphasize standardization, clear documentation, and baseline-ready structure

Dataset Structure Improvements

Based on paper review, implemented the following improvements:

  • Naming Convention: [Event]_[Location]_[Date]_[Type].tif
  • Metadata Schema: JSON files with coordinates, timestamps, disaster type, data source
  • Organization: Separate folders by disaster type → location → temporal phase
  • Mask Format: Binary PNG/GeoTIFF (0=no change, 255=change)
  • Documentation: README files explaining structure and usage

Benefits of Multi-Hazard Approach

  • Models trained on diverse disasters generalize better
  • Dataset suitable for multi-task learning frameworks
  • Increased research community interest and usability
  • Enables disaster-type classification in addition to change detection
  • Broader impact for global disaster monitoring applications
