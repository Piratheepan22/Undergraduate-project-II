Week 5

Overview

This week, We focused on developing a custom annotation tool for creating binary masks to label changed and unchanged areas in the collected image pairs. I created a web-based software that allows uploading pre-disaster and post-disaster images simultaneously, overlaying them with adjustable opacity, and manually annotating changed regions using polygon drawing tools. The tool generates binary masks where white represents unchanged areas and black represents changed/damaged areas, providing essential ground truth labels for training change detection models.

Detailed Update

With a comprehensive dataset of satellite image pairs collected over the past weeks, the next critical step was to create ground truth annotations. These annotations—in the form of binary masks—are essential for supervised machine learning approaches to change detection. Rather than using generic annotation tools, I developed a specialized application tailored to the specific needs of this disaster mapping project.

The Need for Custom Annotation Tool

Change detection annotation requires comparing two images from different time periods and precisely identifying areas where changes have occurred. This task has unique requirements:

  • Simultaneous viewing of before and after images for accurate comparison
  • Ability to toggle between images or overlay them with adjustable transparency
  • Precise polygon drawing tools for marking irregular change boundaries
  • Zoom and pan capabilities for detailed annotation of small features
  • Export functionality to generate binary masks in standard image formats
  • Intuitive interface to speed up the annotation process across many image pairs
Existing general-purpose annotation tools often lack these specialized features, motivating the development of a custom solution.

Annotation Tool Development

We developed a web-based annotation tool with the following key features:

Core Functionality:

  • Dual Image Upload:
    • Upload pre-disaster (before) image
    • Upload post-disaster (after) image
    • Automatic image alignment on the same canvas
  • Image Overlay System:
    • Layer the after image over the before image
    • Adjustable opacity slider to control transparency of the overlay
    • Toggle between images to identify changes easily
    • Smooth transition for visual change identification
  • Polygon Drawing Tools:
    • Click to create polygon vertices around changed areas
    • Close polygons to define change regions
    • Multiple polygon support for multiple change areas in one image
    • Edit and delete capabilities for refining annotations
  • Navigation Controls:
    • Zoom in/out functionality for detailed annotation
    • Pan across the image canvas
    • Reset view to original scale and position
  • Binary Mask Export:
    • Generate binary mask from drawn polygons
    • White pixels (255) for unchanged areas
    • Black pixels (0) for changed/damaged areas
    • Export in standard image formats (PNG, TIFF)
    • Same resolution as input images for pixel-perfect alignment

Technical Implementation Details:

  • Technologies Used:
    • HTML5 Canvas for image rendering
    • JavaScript for interactivity and logic
    • CSS3 for responsive design
    • Canvas API for drawing and manipulation
  • Image Processing:
    • Client-side processing for fast performance
    • Efficient rendering of large satellite images
    • Real-time polygon rendering during drawing
  • Export Format:
    • Binary PNG images (single channel)
    • Pixel value 0 (black) = changed area
    • Pixel value 255 (white) = unchanged area
    • Same dimensions as input images
