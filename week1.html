Detailed Update

The creation of a reliable change detection dataset is essential for training and validating models that can identify temporal changes in land cover, urban expansion, deforestation, and other environmental transformations. This week marked the foundational stage of this dataset development process.

What is Change Detection?

Change detection in remote sensing refers to the process of identifying differences in the state of an object or phenomenon by observing it at different times. In the context of satellite imagery, this involves:

  • Comparing multi-temporal satellite images of the same geographic area
  • Identifying and quantifying changes such as urban growth, vegetation loss, water body expansion, or infrastructure development
  • Creating labeled datasets that can train machine learning models to automatically detect these changes

Google Earth Engine (GEE)

Google Earth Engine is a cloud-based platform for planetary-scale geospatial analysis. It provides:

  • Access to a massive archive of satellite imagery spanning several decades
  • Pre-processed datasets from satellites like Landsat, Sentinel, MODIS, and more
  • JavaScript and Python APIs for data processing and analysis
  • Cloud computing capabilities to handle large-scale geospatial computations
This week, I created my GEE account and explored its interface, code editor, and available satellite image collections.
