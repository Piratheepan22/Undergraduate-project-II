Week 3

Overview

This week, we focused on the technical implementation phase of the dataset creation. Using the list of affected areas compiled last week, we developed Google Earth Engine (GEE) code to systematically collect satellite image pairs showing the conditions before, during, and after the November 2025 disaster. I successfully implemented image filtering, preprocessing, and began collecting image pairs for the identified affected locations across Sri Lanka.

Detailed Update

Building upon the comprehensive location list created last week, this week marked the transition from planning to implementation. The primary goal was to develop robust GEE scripts that could efficiently collect high-quality satellite imagery for all identified disaster-affected locations.

Google Earth Engine Code Development

we developed a comprehensive GEE script with the following key functionalities:

  • Region of Interest (ROI) Definition: Code to define geographic boundaries for each affected area using coordinates
  • Temporal Filtering: Scripts to filter images into three categories:
    • Before disaster: Images from before The disaster
    • During disaster: Images captured around November 28-29, 2025 on the disaster time
    • After disaster: Images from after The disaster
  • Image Compositing: Creating median composites when multiple images are available for the same time period
  • Band Selection: Selecting appropriate spectral bands for visualization and analysis
  • Export Functionality: Code to download processed images for dataset creation

Tasks Completed This Week:

  • Developed complete GEE code for image collection
  • Implemented functions to:
    • Load satellite image collections (Sentinel-2)
    • Filter images by date range and geographic location
    • Generate image pairs for comparison
  • Set up the GEE code editor workspace with organized scripts
  • Tested the code with sample locations from the affected areas list
  • Began systematic collection of image pairs for priority locations
  • Created a systematic workflow for processing each location
  • Organized collected image pairs with proper naming conventions

Image Pair Collection Progress:

we have successfully begun collecting image pairs for the priority locations identified last week. The focus has been on areas where:

  • Clear pre-disaster baseline imagery is available
  • The disaster impact was most severe and visible
  • Post-disaster imagery shows significant changes
  • Cloud cover is minimal across all three time periods
  • Ground features are clearly distinguishable

Sample Image Pairs Collected:

Initial image pairs have been collected showing:

  • Flooding in low-lying urban and agricultural areas
  • Landslide scars in hilly regions
  • Changes in water body extents
  • Infrastructure damage and debris
  • Vegetation changes in affected zones
