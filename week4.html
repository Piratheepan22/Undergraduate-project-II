Week 4

Overview

This week, We continued the systematic collection of satellite image pairs for all affected locations identified in Week 2. Using the GEE code developed in Week 3, We processed and downloaded images for multiple disaster-affected areas across Sri Lanka. All collected images were organized with proper naming conventions and folder structures for efficient dataset management. Additionally, I continued reviewing research papers on change detection methodologies and annotation techniques.

Detailed Update

This week marked a period of intensive data collection. Building on the foundation established in previous weeks, We focused on systematically processing each affected location from the compiled list, ensuring comprehensive coverage of the disaster-impacted areas across Sri Lanka.

Image Collection Progress

Using the GEE code developed last week, We processed satellite imagery for numerous affected locations. The collection process involved:

  • Running the GEE script for each location from the affected areas list
  • Collecting image triplets (before, during, after) for each area
  • Verifying image quality and cloud cover for each collection
  • Downloading and organizing images with proper naming conventions

Tasks Completed This Week:

  • Systematically processed all locations from the affected areas list compiled in Week 2
  • Collected image pairs for multiple districts across Sri Lanka, including:
    • Flood-affected urban and agricultural areas
    • Landslide-prone hilly regions
    • Areas with both flood and landslide impacts
    • Critical infrastructure zones affected by the disaster
  • Continued reading research papers focusing on:
    • Annotation methodologies for change detection datasets
    • Binary mask creation techniques
  • Created a collection tracking spreadsheet to monitor progress
  • Identified challenging locations requiring special processing or alternative approaches

Challenges Encountered:

  • Some locations had persistent cloud cover during the disaster period, requiring multiple date searches
  • Certain remote areas had limited high-resolution imagery available
  • Temporal gaps between available images for some locations
  • Large file sizes requiring efficient storage management
  • Processing time for downloading and organizing large numbers of images

Solutions Implemented:

  • Extended date ranges for cloudy locations to find clearer imagery
  • Used alternative satellite sources (Landsat) for areas lacking Sentinel-2 coverage
  • Implemented automated download scripts to handle large batches efficiently
  • Set up organized file compression for long-term storage
  • Created batch processing workflows to speed up collection

Research Papers Refered

This week's literature review focused on papers that would inform the next phase of work—annotation and labeling. Key topics explored included:

  • Manual vs. automated annotation techniques for satellite imagery
  • Binary mask generation for change detection
  • Interactive annotation tools and platforms
  • Quality assurance protocols for labeled datasets
  • Inter-annotator agreement metrics
  • Best practices for polygon-based annotation
These papers provided valuable insights that will guide the annotation process in the coming weeks.
